# ISC Benchmark Pipeline

Inter-sample consistency (ISC) benchmark pipeline used in the manuscript reproducibility workflow.

## Scope

This pipeline runs 8 perturbation tasks on processed datasets generated by `data_processing/` to evaluate inter-sample consistency (ISC) robustness.

**Perturbation tasks:**

1. **Signal degradation** — Sensitivity to progressive disruption of cell type structure (compactness) in semi-synthetic datasets
2. **Over-partitioning** — Sensitivity to artificial splitting of cell populations into subpopulations
3. **Annotation granularity** — Robustness to cell type annotation coarseness versus fineness (consistency across different levels of detail)
4. **Cellular complexity** — Robustness across tissues with low versus high inter-cell-type variability
5. **Sample size** — Robustness to varying the number of samples
6. **Cell count** — Robustness to varying the number of cells per cell type
7. **Batch effects** — Robustness to systematic technical differences between batches
8. **Biological perturbations** — Robustness to systematic differences induced by biological conditions or experimental perturbations

Dataset participation in each task is controlled by `../data_processing/config/specs_datasets.csv`.

Main entrypoints:
- `_targets.R`
- `scripts/submit_hpc.sh`

## Inputs

Required:
- Processed dataset files in `../data/processed/*.rds`
- Dataset-family mapping in `config/dataset_idents.yaml`
- Pipeline parameters in `config/isc_benchmark_parameters.yaml`

## Quick Start

From repository root:

```bash
Rscript -e 'renv::restore()'
cd ISC_benchmark
Rscript -e 'targets::tar_make()'
```

Resume after interruption:

```bash
cd ISC_benchmark
Rscript -e 'targets::tar_make()'
```

## HPC Execution

Submit SLURM jobs with the provided wrapper:

```bash
cd ISC_benchmark
bash scripts/submit_hpc.sh
```

Useful filters:

```bash
ISC_DATASET_ID="JoaI_CRC-SG1_Normal" bash scripts/submit_hpc.sh
ISC_DATASET_IDS="JoaI_CRC-SG1_Normal,Stephenson_Cambridge_Covid" bash scripts/submit_hpc.sh
ISC_DATASET_FAMILIES="BCC,ICBAtlas" bash scripts/submit_hpc.sh
```

The script can submit stem-level and family-level runs depending on selected tasks and available data.

## Outputs

Primary aggregate output:
- `results/all_results.csv`

Per-task outputs are saved under:
- `results/<dataset_or_family>/<task>_<ident>/`

Each persisted run can include:
- `<dataset>_<task>_<ident>_metrics.rds`
- `<dataset>_<task>_<ident>_metadata.yaml`

## Configuration

Edit `config/isc_benchmark_parameters.yaml`:
- `run`: enable/disable tasks
- `common`: shared benchmark settings
- `task_*`: task-specific settings

Edit `config/dataset_idents.yaml` to define valid dataset families/idents.

## Troubleshooting

- No datasets detected: ensure `../data/processed/*.rds` exists.
- Missing families: verify `config/dataset_idents.yaml` includes your dataset stems.
- HPC module issues: adapt module load lines in `scripts/submit_hpc.sh` for your cluster.
