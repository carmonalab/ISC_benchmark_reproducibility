# Data Processing Pipeline

Preprocessing pipeline that converts raw single-cell datasets into standardized processed datasets used by all downstream benchmarks and figure notebooks.

## Scope

This module prepares the core processed inputs consumed by:
- `ISC_benchmark/`
- `label_transfer_task/`
- `Figures_notebooks/`

Main entrypoints:
- `_targets.R`
- `scripts/master_job.sh`
- `scripts/submit_hpc.sh`

## Inputs

Required raw files must be downloaded from Zenodo DOI `10.5281/zenodo.18921437` into:
- `../data/raw/`

Dataset registry and processing parameters:
- `config/core_datasets.yaml`
- `config/processing_parameters.yaml`

## Quick Start

From repository root:

```bash
Rscript -e 'renv::restore()'
cd data_processing
Rscript -e 'targets::tar_make()'
```

Resume after interruption:

```bash
cd data_processing
Rscript -e 'targets::tar_make()'
```

Check status:

```bash
cd data_processing
Rscript -e 'targets::tar_status()'
```

## HPC Execution

Run via wrapper:

```bash
bash data_processing/scripts/master_job.sh
```

Submit to SLURM:

```bash
sbatch data_processing/scripts/submit_hpc.sh
```

## Outputs

Primary outputs:
- Processed datasets in `../data/processed/`
- Per-dataset metadata in `../data/processed/*.yaml` (when generated by processing helpers)

These processed files are the required inputs for downstream pipeline READMEs.

## Configuration

Edit `config/processing_parameters.yaml` for:
- random seed
- downsampling limits
- sample/cell filtering thresholds
- parallel settings

Edit `config/core_datasets.yaml` for:
- dataset IDs
- raw file mappings
- metadata column mapping per dataset

## Troubleshooting

- No input files found: verify `.rds` files exist in `../data/raw/`.
- Unexpected skips/failures: inspect target status and rerun to resume.
- HPC environment errors: adapt module loads and SBATCH headers in `scripts/submit_hpc.sh`.
